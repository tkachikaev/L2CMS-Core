# KaevCMS на обычном хостинге

1. Загрузите полный архив KaevCMS. Для web-установки внутри должна присутствовать папка `vendor/`.
2. В панели хостинга укажите корнем домена каталог `public/`.
3. Подключите SSL-сертификат и открывайте установщик по HTTPS.
4. Откройте сайт. Если файла `.env` ещё нет, KaevCMS перенаправит на `/install/`.
5. Установщик проверит PHP 8.3+, расширения, права на запись и наличие зависимостей.
6. Укажите адрес сайта и реквизиты MySQL. Проверка создаст и удалит служебную таблицу, чтобы подтвердить реальные права пользователя базы.
7. Создайте владельца CMS и завершите установку.

После успешной установки создаётся `storage/app/installed.lock`, а повторный запуск мастера блокируется. Если установка оборвалась после создания `.env`, миграций или владельца, `storage/app/installing.lock` позволяет продолжить процесс без смены `APP_KEY` и без создания второго владельца.

Пароль MySQL не возвращается в HTML. Он хранится только в серверной сессии установщика и затем в `.env`. Внутренние ошибки записываются в `storage/logs/installer.log` с коротким кодом, но не показывают SQL, пути и пароль в браузере.

Web Installer не запускает Composer, npm или shell-команды. На текущем этапе исходный ZIP поставляется без `vendor`; для теста на хостинге скопируйте `vendor` из локальной установленной KaevCMS 0.31.9. Отдельный Hosting Edition с production-папкой `vendor` будет собираться позже.

Apache использует `public/.htaccess`. Если хостинг не позволяет указать `public/` корнем домена, не переносите всё приложение в публичный каталог: используйте поддомен или настройку пользовательского Document Root.
