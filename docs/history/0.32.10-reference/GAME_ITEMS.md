# Названия и иконки игровых предметов

KaevCMS не хранит переводы названий предметов внутри промокодов. Один общий каталог используется веб-инвентарём, историей передач, очередью наград, промокодами и будущими модулями.

## Каталог названий

Для каждого установленного языка можно добавить или изменить файл:

```text
lang/{locale}/items.php
```

Встроенные примеры:

```text
lang/ru/items.php
lang/en/items.php
```

Формат:

```php
<?php

return [
    'common' => [
        57 => 'Адена',
        4037 => 'Монета удачи',
    ],

    'servers' => [
        3 => [
            90000 => 'Особая монета сервера',
        ],
    ],
];
```

`common` содержит общие названия. В `servers` ключом служит ID GameServer из KaevCMS, а его значения переопределяют общий каталог только для выбранного сервера.

Для английского языка создаётся такой же набор с английскими названиями:

```php
return [
    'common' => [
        57 => 'Adena',
        4037 => 'Coin of Luck',
    ],
];
```

Каталог необязателен для работы очереди: `item_id` и количество всегда передаются независимо от названия. Однако неизвестный предмет показывается игроку как нейтральный «Игровой предмет» без технического ID. В административных журналах ID остаётся видимым мелкой технической подписью.

## Порядок поиска названия

KaevCMS проверяет:

1. серверное переопределение текущего языка;
2. общий каталог текущего языка;
3. серверное переопределение fallback-языка;
4. общий каталог fallback-языка;
5. сохранённое legacy-название награды, если оно есть;
6. нейтральное название без номера предмета.

Изменение каталога применяется и к уже созданным наградам: миграция базы не требуется.

## Иконки

Иконки остаются независимыми от текста и загружаются по ID:

```text
public/uploads/game-assets/items/common/57.webp
public/uploads/game-assets/items/servers/3/57.png
```

Серверная иконка имеет приоритет над общей. Поддерживаются `webp`, `png`, `jpg` и `jpeg`.
