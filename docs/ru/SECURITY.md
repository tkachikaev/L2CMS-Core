# Безопасность и права файлов

## Граница публичного доступа

Через веб-сервер должен быть доступен только подготовленный публичный каталог. `.env`, `vendor`, код приложения, журналы, резервные копии базы и журнал установщика должны находиться в закрытом ядре.

## Рекомендуемые права

Типовые значения для Linux:

```text
Обычные файлы                         0644
Обычные каталоги                      0755
storage/                              0755 или 0775
bootstrap/cache/                      0755 или 0775
публичные uploads/                    0755 или 0775
.env                                  0600 или 0640
storage/app/installed.lock            0600 или 0640
```

Не назначайте `0777` рекурсивно всему проекту. KaevCMS проверяет фактическую запись и показывает каталоги с world-writable правами как предупреждение.

Резервные копии Web/CLI Updater на Linux создаются в каталогах `0700`, а SQL-дампы и metadata получают `0600`. Публичный каталог `uploads` содержит Apache-защиту от выполнения PHP-подобных файлов; в nginx разрешайте PHP только для `index.php` и `install/index.php`.

Права являются только одним уровнем защиты. На хостингах с ACL и изоляцией пользователей цифровой режим может не полностью отражать реальный доступ. Поэтому Web Installer показывает и тест записи, и доступные POSIX-права.

## Итоговая проверка Web Installer

После установки мастер проверяет:

- расположение закрытого ядра;
- расположение и права `.env`;
- значение `APP_DEBUG=false`;
- HTTPS и защищённые cookie;
- доступность runtime-каталогов на запись;
- права публичных загрузок;
- наличие `installed.lock`.

Критические проблемы отмечаются красным, рекомендации — жёлтым.

## Правила эксплуатации

- Включите HTTPS до ввода паролей базы и владельца.
- Используйте отдельную пустую базу и отдельного пользователя MySQL.
- Оставляйте только production-зависимости.
- Не публикуйте ZIP релизов, резервные копии, `.env` и журналы установщика.
- Проверяйте аудит и runtime-диагностику.
- Запускайте аудит зависимостей при наличии доступа в интернет.
