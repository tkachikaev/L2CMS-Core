# Обновления

KaevCMS выпускается в трёх форматах:

- полный архив исходников;
- Windows-патч с предыдущей версии;
- кумулятивный Update ZIP для поддерживаемого диапазона версий.

Один кумулятивный пакет может обновить, например, `0.32.5` сразу до `0.32.15`. Устанавливать все промежуточные версии не требуется, если текущая версия входит в диапазон пакета.

## Перед обновлением

1. Сохраните `.env`, базу CMS, публичные uploads и собственные изображения.
2. Сверьте SHA256 архива с опубликованной контрольной суммой.
3. Загружайте пакеты только из доверенного источника. Текущие пакеты проверяют целостность manifest и файлов, но пока не имеют отдельной криптографической подписи издателя.
4. Не заменяйте runtime-секреты и `storage` файлами из полного архива.

## Shared-hosting

Используйте Web Updater в административной панели. Он проверяет диапазон версий, manifest, SHA256 каждого файла, запрещённые пути, свободное место, резервные копии и состояние восстановления.

Web Updater должен работать только при наличии прав на изменение файлов установки. Не назначайте `0777` всему проекту только ради прохождения preflight.

## Ubuntu VDS

На VDS исходный код принадлежит SSH/deployment-пользователю, а PHP-FPM получает запись только в runtime-каталоги. Поэтому обновление применяется через CLI от владельца проекта:

```bash
cd /var/www/kaevcms
php artisan kaevcms:update /tmp/KaevCMS-update.zip
```

Команда показывает проверки и запрашивает подтверждение. Для автоматизации после собственной проверки пакета:

```bash
php artisan kaevcms:update /tmp/KaevCMS-update.zip --yes
```

Не запускайте команду от `www-data` и не выдавайте PHP-FPM запись на весь исходный код.

## Изменение зависимостей

Если релиз меняет `composer.lock`, используйте полный релиз и выполните:

```bash
composer install --no-dev --optimize-autoloader --no-interaction
```

Не используйте `composer update` на production-сервере.

## Windows

После распаковки патча запустите версионный apply-скрипт, затем:

```powershell
.\deployment\windows\quality.ps1
.\deployment\windows\browser-quality.ps1
```
