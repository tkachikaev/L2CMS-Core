# Аудит и стабилизация KaevCMS 0.29.0

Версия 0.29.0 не добавляет новый крупный модуль. Её задача — проверить релиз после изменений модулей, промокодов, веб-инвентаря, очереди наград и общего драйвера L2J Mobius, удалить тестовые остатки и закрепить обнаруженные проблемы регрессионными проверками.

## Исправлено

### Светлый личный кабинет Kaev Aurelia Account

- блок последних активаций промокодов переведён на общую округлую поверхность `account-surface`;
- элементы веб-инвентаря используют единые токены радиусов вместо разрозненных значений;
- декоративная рамка боковой карточки формы теперь привязана к самой карточке через `position: relative` и обрезается её скруглением;
- маленькие изображения предметов в истории больше не отображаются квадратными внутри округлого интерфейса;
- активный пункт установленного модуля обновляется через `wire:current` после Livewire-навигации в сохранённой боковой панели;
- удалены два неработающих CSS-селектора для псевдоэлементов, которых не существовало.

Версия темы повышена до 1.1.2.

### Безопасность журналов

Из рабочих логов GameServer-адаптера и мониторинга онлайна убран полный текст исключения внешней базы. Для диагностики сохраняются операция, ID сервера, драйвер, класс исключения и безопасный код, но внутренний текст PDO/MySQL больше не попадает в журнал приложения.

### Очистка релиза

Из чистой сборки удалён `integrations/reward-queue/remove-legacy-bridge.sql`. Это был одноразовый файл для ранних тестов отменённого Reward Bridge. Исторические записи CHANGELOG сохранены, потому что они документируют развитие проекта.

Системные heartbeat Laravel Scheduler и database-очередей не относятся к Reward Bridge и остаются в CMS: они контролируют почту и фоновые задания ядра.

## Проверяемые сценарии

Добавлены или усилены проверки:

- manifest и версия Kaev Aurelia Account;
- наличие единой округлой поверхности на странице промокодов;
- безопасная активная ссылка модуля после SPA-навигации;
- вычисленные браузером скругления страницы промокодов и веб-инвентаря;
- правильное позиционирование и обрезка декоративного слоя формы;
- отсутствие legacy SQL-файла Bridge в релизе;
- отсутствие полного текста исключения внешней БД в рабочих логах.

## Статические проверки релиза

Перед упаковкой должны пройти:

- синтаксис всех PHP-файлов;
- синтаксис JavaScript и browser-spec файлов;
- корректность JSON manifest и языковых файлов;
- полный паритет RU/EN;
- отсутствие временных `.bak`, `.orig`, `.rej`, `.tmp` файлов;
- отсутствие рабочего кода Reward Bridge;
- совпадение полной сборки 0.29.0 и результата наложения патча на 0.28.0.

Полные Pint, PHPStan, PHPUnit и Playwright выполняются на установленном проекте командами `quality.ps1` и `browser-quality.ps1`.

## Следующий функциональный этап

После принятия стабилизационной версии следующим крупным релизом планируется модуль Pally Donations с идемпотентным журналом платежей, проверкой входящих уведомлений и начислением наград в общий веб-инвентарь.
