# Администраторы

Раздел управления администраторами находится по адресу:

```text
/admin/administrators
```

Все администраторы L2Forge CMS имеют одинаковые права доступа. Роли и отдельные разрешения в текущей версии не используются, поскольку панель обычно обслуживают один или два человека.

## Список администраторов

В списке отображаются:

- имя;
- email для входа;
- дата создания;
- дата последнего успешного входа;
- состояние двухфакторной аутентификации;
- состояние учётной записи;
- действия изменения, включения или отключения.

Для администратора, который ещё не входил в панель, выводится значение `Никогда`.

## Создание администратора

Нажмите **«Создать администратора»** и заполните:

- имя;
- уникальный email;
- пароль;
- подтверждение пароля.

Email автоматически приводится к нижнему регистру и используется для входа в административную панель.

Требования к паролю:

- не менее 12 символов;
- минимум одна строчная буква;
- минимум одна заглавная буква;
- минимум одна цифра.

Пароль хранится только в виде Argon2id-хэша согласно настройкам Laravel.

## Изменение данных и пароля

На странице администратора можно изменить имя и email.

Для смены собственного пароля необходимо указать текущий пароль. При изменении пароля другого администратора текущий пароль инициатора не запрашивается.

Пароли и их хэши никогда не сохраняются в журнале действий.


## Двухфакторная аутентификация

Каждый администратор подключает 2FA самостоятельно через меню пользователя **«Безопасность аккаунта»**:

```text
/admin/account/security
```

Поддерживается стандартный TOTP-код из приложения-аутентификатора и восемь одноразовых резервных кодов. Секрет хранится зашифрованным через `APP_KEY`, а резервные коды — только в виде хешей.

В списке и карточке администратора видны только статус и дата подключения. Другой администратор не может увидеть секрет, QR-код или резервные коды и не может включить 2FA за владельца аккаунта.

Для чужой учётной записи доступен аварийный сброс после ввода собственного текущего пароля. Сброс удаляет данные 2FA и отзывает активные сессии целевого администратора. Свою 2FA администратор отключает только на странице безопасности аккаунта, подтвердив пароль и действующий TOTP либо резервный код.

При потере доступа владелец сервера может выполнить:

```powershell
php artisan l2forge:admin-2fa:disable admin@example.com
```

## Отключение

Физическое удаление администраторов через веб-интерфейс не предусмотрено. Учётную запись можно отключить, чтобы сохранить связь с историей действий.

Действуют ограничения:

- нельзя отключить собственную учётную запись;
- нельзя отключить последнего активного администратора;
- отключённый администратор не может войти;
- уже открытая сессия отключённого администратора прекращается при следующем запросе к панели.

Отключённую учётную запись можно включить повторно без создания нового администратора.

## Журнал действий

В журнал записываются:

- создание администратора;
- изменение имени или email;
- смена пароля без сохранения самого пароля;
- включение, проверка, отключение и сброс 2FA;
- использование и регенерация резервных кодов без сохранения самих кодов;
- включение;
- отключение.

Подробности журнала: [AUDIT_LOG.md](AUDIT_LOG.md).

## Первый администратор

При чистой установке первый администратор создаётся командой:

```powershell
php artisan l2forge:admin-create
```

После этого дополнительные учётные записи удобнее создавать через административную панель.
