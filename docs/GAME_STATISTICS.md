# Публичная игровая статистика

## Назначение

CMS не обращается к таблицам конкретной сборки из контроллеров и шаблонов. Каждый игровой драйвер реализует единый read-only контракт `GameWorldDriver`, а публичная страница получает уже нормализованные данные.

Первая реализация — `L2J Mobius CT0 Interlude` (`l2j_mobius_ct0_interlude`).

## Поддерживаемые разделы

- рейтинг по уровню;
- рейтинг PvP;
- рейтинг PK;
- рейтинг по общему времени в игре;
- текущие герои;
- владельцы замков.

Крепости в Interlude не поддерживаются и не выводятся.

## Используемые таблицы

### `characters`

Используются только игровые сведения персонажа:

- `charId`;
- `char_name`;
- `level` и `exp`;
- `classid`;
- `race` и `sex`;
- `title`;
- `online`;
- `lastAccess`;
- `onlinetime`;
- `pvpkills`, `pkkills`, `karma`;
- `nobless`;
- `clanid`;
- `accesslevel` и `deletetime` для исключения служебных и удалённых персонажей.

### `clan_data`

Используются `clan_id`, `clan_name`, `clan_level`, `reputation_score`, `leader_id` и `hasCastle`.

### `heroes`

Наличие строки с `charId` определяет текущего героя.

### `castle`

`clan_data.hasCastle` связывается с `castle.id`, после чего CMS показывает название замка, клан и лидера.

## Приватность

Публичные запросы не выбирают и не передают в шаблон:

- логин игрового аккаунта;
- пароль или его хеш;
- email;
- IP и HWID;
- координаты;
- инвентарь и склад;
- платёжные и донатные данные.

Персонажи с `accesslevel != 0` и записи с `deletetime != 0` исключаются из рейтингов. Глобальная настройка скрытия публичного онлайна также убирает статус онлайн из рейтинга уровня.

## Нагрузка

Успешные выборки кешируются на пять минут отдельно по игровому миру, разделу, лимиту и языку. При ошибке подключения для той же выборки включается 30-секундный cooldown, чтобы публичный трафик не создавал повторные соединения с недоступной игровой базой.

Данные не копируются в базу CMS. Исторические графики и снимки онлайна будут отдельным этапом.
