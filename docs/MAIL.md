# Почтовая система

Почтовые настройки доступны администратору:

```text
/admin/settings/mail
```

Раздел состоит из внутренних вкладок:

```text
Подключение
Подтверждение email
Восстановление пароля
Пароль изменён
```

## Подключение SMTP

Во вкладке «Подключение» настраиваются:

```text
SMTP-сервер
Порт
STARTTLS / TLS, SSL / SMTPS или соединение без принудительного шифрования
Имя пользователя SMTP
Пароль SMTP
Email отправителя
Имя отправителя
Email для системных уведомлений
```

Имя пользователя и пароль можно оставить пустыми, если почтовый сервер не требует авторизацию.

После сохранения необходимо отправить тестовое письмо. Любое изменение параметров SMTP сбрасывает результат предыдущей проверки. До новой успешной проверки нельзя включить регистрацию с обязательным подтверждением email.

## Шаблоны писем

L2Forge CMS содержит готовые стандартные шаблоны. После установки ничего настраивать вручную не требуется.

Администратор может изменить:

```text
Тему письма
Заголовок
Основной текст
Текст кнопки
Дополнительный текст
```

HTML-каркас письма, адаптивная вёрстка и реальная ссылка кнопки формируются CMS автоматически. HTML-теги в полях шаблона запрещены. Для простого выделения допускается Markdown:

```text
**жирный текст**
*курсив*
```

Новый абзац создаётся пустой строкой.

## Доступные переменные

Переменные записываются в двойных фигурных скобках:

```text
{{site_name}}
{{site_url}}
{{username}}
{{user_email}}
{{support_email}}
```

Для подтверждения email также доступны:

```text
{{verification_url}}
{{expires_in}}
```

Для восстановления пароля:

```text
{{reset_url}}
{{expires_in}}
```

Неизвестные переменные не сохраняются. Реальные токены и ссылки создаются самой CMS и не показываются администратору.

## Стандартные значения

Стандартные шаблоны хранятся в коде:

```text
config/mail_templates.php
```

Пользовательские изменения сохраняются в таблице `cms_settings`. Обновление CMS не перезаписывает изменённые тексты.

Кнопка «Восстановить стандартный шаблон» удаляет пользовательское переопределение и возвращает заводской вариант из `config/mail_templates.php`.

## Предпросмотр и тестовая отправка

Каждая вкладка шаблона содержит:

```text
Предпросмотр
Отправить тестовый шаблон
```

Предпросмотр использует демонстрационные значения. Тестовая отправка создаёт безопасную примерную ссылку и не выпускает настоящий токен подтверждения или восстановления.

Перед тестовой отправкой шаблона SMTP должен быть настроен и проверен во вкладке «Подключение».

## Уведомление после смены пароля

После успешной установки нового пароля CMS отправляет отдельное уведомление. Ошибка этого уведомления не отменяет уже выполненную смену пароля, но фиксируется в журнале.

## Хранение SMTP-пароля

Пароль SMTP:

- шифруется средствами Laravel с использованием `APP_KEY`;
- не выводится обратно в HTML;
- не записывается в журнал;
- не изменяется, если при повторном сохранении поле пароля оставлено пустым.

Если `APP_KEY` потерян или заменён, старый SMTP-пароль расшифровать нельзя. Потребуется ввести его заново.

## Журналирование

Журнал фиксирует:

```text
mail.template_updated
mail.template_reset
mail.template_test_sent
mail.template_test_failed
mail.password_changed_sent
mail.password_changed_failed
```

Полное содержимое шаблонов, реальные ссылки и токены в журнал не записываются. Для изменения шаблона сохраняются только его название и перечень изменённых полей.

## DNS и адрес сайта

Для реального домена рекомендуется настроить SPF, DKIM и DMARC у почтового провайдера.

`APP_URL` должен содержать настоящий публичный адрес сайта. Он используется при формировании подписанных ссылок подтверждения email и восстановления пароля.

## Локальная разработка

До настройки SMTP применяется безопасный почтовый транспорт `log`. Регистрация с обязательным подтверждением email недоступна, пока администратор не сохранит SMTP и не выполнит успешную тестовую отправку.

## Язык письма

Шаблоны подтверждения email, восстановления и уведомления о смене пароля имеют отдельную версию для каждого включённого языка. Русский и английский заводские варианты готовы сразу после установки.

Язык письма берётся из `users.locale`. Подписанные ссылки и ссылки восстановления ведут на локализованные маршруты. Для дополнительного языкового пакета администратор может заполнить свой вариант шаблона; пока он не заполнен, применяется резервный язык.

Ключи пользовательских шаблонов имеют вид:

```text
mail.template.email_verification.en.subject
mail.template.password_reset.ru.body
```
