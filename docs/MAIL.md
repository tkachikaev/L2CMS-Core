# Почтовая система

Почтовые настройки доступны администратору:

```text
/admin/settings/mail
```

Версия 0.8.0 поддерживает SMTP.

## Параметры

```text
SMTP-сервер
Порт
STARTTLS / TLS, SSL / SMTPS или без принудительного шифрования
Имя пользователя SMTP
Пароль SMTP
Email отправителя
Имя отправителя
Email для будущих системных уведомлений
```

Имя пользователя и пароль можно оставить пустыми, если почтовый сервер не требует авторизацию.

## Хранение пароля

Пароль SMTP:

- шифруется средствами Laravel с использованием `APP_KEY`;
- не выводится обратно в HTML;
- не записывается в журналы;
- не изменяется, если при повторном сохранении поле пароля оставлено пустым.

Если `APP_KEY` потерян или заменён, старый пароль SMTP расшифровать нельзя. Потребуется ввести его заново.

## Проверка

После сохранения необходимо отправить тестовое письмо. Успешная дата проверки хранится в `cms_settings`.

Любое изменение почтовых параметров сбрасывает статус проверки. До новой проверки нельзя включить регистрацию с обязательным подтверждением email.

## DNS и адрес сайта

Для реального домена рекомендуется настроить SPF, DKIM и DMARC у почтового провайдера.

`APP_URL` должен содержать настоящий публичный адрес сайта. Он используется при формировании подписанных ссылок подтверждения email и восстановления пароля.

## Локальная разработка

До настройки SMTP применяется безопасный почтовый транспорт `log`. Реальная регистрация с обязательным подтверждением email при этом недоступна, пока администратор не сохранит SMTP и не выполнит тестовую отправку.
